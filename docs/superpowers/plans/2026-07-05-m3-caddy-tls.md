# M3: Caddy + TLS Implementation Plan

> Executed 2026-07-05. Two steps were amended during execution: the Caddyfile ships with no auto_https block (Task 1 gate; ADR 0018 M3 execution note), and Caddy snippets flattened to `/etc/caddy/Caddyfile.d/<service>-<env>.caddy` with import glob `*.caddy` (Caddy import globs allow one wildcard). If replaying Task 8 Step 6, write the flat snippet path, not `Caddyfile.d/hello/prod.caddy`. See `docs/setup/m3-infra-state.md` for the as-built record.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Serve `https://hello.wingkongexchange.dev` with a valid wildcard certificate through Cloudflare, from a Compose-run Caddy on the M2 Host, with the origin unreachable except through Cloudflare.

**Architecture:** Layer 3 lands as the Platform stack (`platform-prod` Compose project) running a custom xcaddy-built Caddy that issues one DNS-01 wildcard cert for `*.wingkongexchange.dev` and imports per-project host-block snippets (ADR 0018). Apps reach Caddy over the shared `wkx-edge` network using `<service>-<env>` aliases (ADR 0019). The `hello` mini-app proves the whole path. Everything is specified in `docs/superpowers/specs/2026-07-05-m3-caddy-tls-design.md`.

**Tech Stack:** Terraform (aws + cloudflare v5 providers), Docker Compose, Caddy 2 + caddy-dns/cloudflare via xcaddy, Python 3.13 stdlib, SSM Session Manager.

## Global Constraints

- `env` is always explicit, never defaulted. Compose files use `${ENV:?}` required-variable syntax so an unset env is a hard error.
- No SSH; all box access via `aws ssm start-session` (ADR 0003).
- Committed files never carry real account state (Invariant 7): no AWS account IDs, zone IDs, EIPs, or IPv6 addresses. Real values go in gitignored `*.local.*` files or on-box gitignored `.env` files.
- ARM64 images only (`--platform linux/arm64`); the Mac builds them natively.
- AWS resources carry `default_tags` plus `Name`; per-service resources add `Service` (and `Env` where the resource has an env dimension).
- Terraform: `terraform fmt`, `terraform validate`, and `terraform test` (aws root) must pass before every apply. Never hardcode an AMI or account ID.
- Prose in NZ English, no em dashes. Diagrams in mermaid.
- Commit style: `infra(m3):`, `feat(m3):`, `docs(m3):` matching repo history. Work on a `feat/m3-caddy-tls` branch cut from `main` after the design branch merges.
- ADR 0017: any `host/cloud-init.yaml` change replaces the Host. Task 3 does this deliberately, once.
- The two hands-on artefacts that define done: `https://hello.wingkongexchange.dev` returns 200 with valid TLS; `curl` direct to the EIP is blocked.

- [ ] **Step 3: Write `hello/src/app.py`**

```python
"""hello: the M3 smoke-test app and ancestor of the reference project (M8).

Responds 200 to every GET with MESSAGE from the environment, so M5 can
change the page by setting /wkx/hello/<env>/MESSAGE and redeploying.
"""
import os
from http.server import BaseHTTPRequestHandler, ThreadingHTTPServer

MESSAGE = os.environ.get("MESSAGE", "hello, wing kong exchange")
PORT = 8000


class Handler(BaseHTTPRequestHandler):
    def do_GET(self):
        body = f"<!doctype html>\n<title>hello</title>\n<h1>{MESSAGE}</h1>\n".encode()
        self.send_response(200)
        self.send_header("Content-Type", "text/html; charset=utf-8")
        self.send_header("Content-Length", str(len(body)))
        self.end_headers()
        self.wfile.write(body)

    def log_message(self, fmt, *args):
        # BaseHTTPRequestHandler logs to stderr; keep that, docker captures it.
        super().log_message(fmt, *args)


if __name__ == "__main__":
    ThreadingHTTPServer(("0.0.0.0", PORT), Handler).serve_forever()
```
