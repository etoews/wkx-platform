# Region ap-southeast-2 (Sydney)

Status: accepted

The platform runs in ap-southeast-2 (Sydney). For interactive, personal use from New Zealand, lower latency outweighs the roughly 10 to 15% cost premium over a cheaper region such as us-east-1. Single region is the deliberate scope (see ADR-0002); this ADR records which region and why.

_Source: design spec §8.1, §8.3._
